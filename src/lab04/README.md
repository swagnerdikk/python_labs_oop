# ЛР-4 - Интерфейсы и абстрактные классы (ABC)

## 1. Цель работы

- Изучить абстрактные базовые классы (`ABC`) и интерфейсный подход
- Задать обязательные методы для классов через контракт
- Закрепить полиморфизм через единый интерфейс
- Применить интерфейсы в архитектуре коллекции из ЛР-2

## 2. Описание интерфейсов

Реализованы 2 интерфейса в `interfaces.py`:

- `Printable`
  - обязательный метод: `to_string()`
  - назначение: единый способ вывода объекта
- `Comparable`
  - обязательный метод: `compare_value()`
  - назначение: единый числовой критерий для сортировки/сравнения

## 3. Реализация в классах

В `models.py` интерфейсы реализованы в двух классах:

- `CityBus` (наследник `lab03.models.CityBus`)
  - реализует `Printable.to_string()`
  - реализует `Comparable.compare_value()` через выручку `calculate()`
- `ExpressBus` (наследник `lab03.models.ExpressBus`)
  - реализует `Printable.to_string()`
  - реализует `Comparable.compare_value()` через расчет скорости `calculate()`

Поведение отличается:
- у `CityBus` критерий сравнения — выручка
- у `ExpressBus` критерий сравнения — рекомендуемая скорость
